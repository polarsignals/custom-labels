# Decision-making and Rights of Maintainers

Any maintainer may unilaterally commit changes to this repository (except to this file). Code reviews are recommended
for major changes, but not required.

In case of disagreement that cannot be resolved via consensus, a vote of maintainers shall be taken.

Any changes to this file shall require a majority vote of maintainers (which can be expressed e.g. via approvals on a pull request).

Any maintainer may release a new version of the libraries in this package to crates.io, NPM, or similar package repositories, unless another maintainer objects, in which case a vote shall be taken.

# New Maintainers

Maintainers may be added or removed by changing this file according to the process described above. However, a person under consideration for removal of maintainership shall not be allowed to vote on their own removal.

Any maintainer may unilaterally resign; in such a case, no vote is needed.
